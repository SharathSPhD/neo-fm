# Phase 2 — smoke handoff

Phase 2 code, tests, and the deterministic golden request fixture are
complete on `phase/2b-cocomposer-western`. The Ralph-Wiggum gate
requires a real `demos/phase-2.wav` reproducible from
`scripts/build-demo.sh phase-2`. That step inherits the Phase 1 DGX
bring-up — see `demos/phase-1-SMOKE-HANDOFF.md`.

## What this phase ships without the WAV

- `packages/song-doc/src/index.ts` adds an optional `tags: string[]` to
  `Section`. Re-exported through codegen so the pydantic mirror picks it
  up automatically.
- `packages/co-composer/src/western.ts` —
  `WesternCoComposer.elaborate()` maps a SongDocument into per-section
  synthesis tags: `section:<type>`, `key:<C|G|...>`,
  `progression:<I-V-vi-IV chord names>`, `style:western`,
  `tempo:<ballad|mid-tempo|upbeat|dance>`, `time_sig:`,
  `lead_vocal:`, `texture:`, `instrument:`.
- `packages/co-composer/src/phase2.ts` translates the elaborated
  SongDocument into a `GenerateRequest` whose shape matches
  `docs/contracts/openapi-dgx.yaml`. The translator lives under `src/`
  so the test suite can call it in-process.
- `demos/phase-2-request.golden.json` — the deterministic
  `GenerateRequest` that the Phase 2 demo POSTs to `/v1/generate`. It is
  regenerated by `pnpm --filter @neo-fm/co-composer build-phase-2-request`
  and pinned by `phase2.test.ts` (byte-for-byte).
- `scripts/build-demo.sh phase-2` — runs the regen + drift check, signs
  the request with HMAC, POSTs to `MUSIC_INFERENCE_URL`, copies the WAV
  out of the container, and runs `ffprobe`.
- 11 vitest cases on `@neo-fm/co-composer` (9 western, 2 phase-2
  golden) + the Phase 1 + song-doc tests are still green.

## Operator steps

1. **Complete Phase 1 bring-up first** so that the
   `music-inference` container is running with a loaded model. See
   `demos/phase-1-SMOKE-HANDOFF.md` if not.

2. **Reproduce the WAV**

   ```bash
   export MUSIC_INFERENCE_URL=http://localhost:8000
   export MUSIC_INFERENCE_HMAC_SECRET=$(grep '^MUSIC_INFERENCE_HMAC_SECRET=' infra/.env.dgx | cut -d= -f2)
   scripts/build-demo.sh phase-2
   ```

   The script:
   - re-runs `pnpm --filter @neo-fm/co-composer build-phase-2-request`
     and aborts if the golden file drifted
   - reads `demos/phase-2-request.golden.json` as the request body
   - signs `body + "\n" + timestamp` with HMAC-SHA256
   - POSTs to `${MUSIC_INFERENCE_URL}/v1/generate`
   - copies `combined_file_path` out of the container to
     `demos/phase-2.wav`
   - runs `ffprobe` to confirm duration

3. **Commit and merge**

   ```bash
   git add demos/phase-2.wav
   git commit -m "phase(2): real western-pop demo, regenerable from golden request"
   git push origin phase/2b-cocomposer-western
   gh pr create --base main --fill
   ```
